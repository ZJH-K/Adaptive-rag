# 任务 07：准备内置文档并完成 Day 2 对比验收

> 项目：Adaptive RAG  
> 阶段：Day 2 — 基础问答与结构感知切分  
> 建议执行顺序：7 / 7  
> 前置任务：任务 01 至任务 06全部完成  
> 预计单次任务规模：中等，作为 Day 2 收口任务

## 必须阅读

1. 项目技术文档：`adaptive_rag_project_technical_spec.md`
2. 当前仓库根目录的 `AGENTS.md`（如存在）
3. 与本任务直接相关的现有源码和测试
- AnyKB：无需读取；测试文档应服务于本项目的 Chunk 策略和问答验收。

> 不要为了“熟悉项目”无边界浏览整个 AnyKB 仓库。只有任务明确要求时，才阅读指定文件。

## 目标

准备 5 个适合技术问答和 Chunk 策略对比的内置文档，完成 Recursive 与结构感知 Chunk 的手工/自动对比，并执行 Day 2 全链路验收，形成可审查的证据记录。

## 上下文

技术文档要求 Day 2：

- 准备 5 个内置技术文档；
- 同一问题可使用两种 Chunk 策略回答；
- PDF 引用包含页码；
- Markdown 引用包含章节；
- 至少记录一组结构感知切分优于固定切分的案例。

本任务不再新增核心算法，重点是构造可复现样例、运行链路、补齐必要测试和记录结果。

## 范围

### 必须实现

- 在 `knowledge/markdown/` 和 `knowledge/pdf/` 下准备总计 5 个技术文档。
- 文档内容应覆盖：
  - 多级 Markdown 标题；
  - 相似章节名但不同语义；
  - 专有名词、函数名或配置项；
  - 至少一个 2 页以上 PDF；
  - 至少一个跨页相关但可按页引用的问题。
- 文档必须来源合规：
  - 优先自行编写小型测试文档；
  - 或使用允许再分发的公开资料并记录来源/许可证。
- 为每个文档设计至少 2 个问题，形成最小验收问题集。
- 使用 `recursive` 与优化策略分别入库或在隔离 collection 中运行。
- 记录至少一组明确案例，证明结构感知切分在以下至少一项更好：
  - 召回 Chunk 章节更完整；
  - 引用定位更准确；
  - 答案上下文噪声更少；
  - 相同 top_k 下相关信息排名更靠前。
- 执行 Day 2 全量测试。
- 生成 `docs/day2_acceptance_report.md` 或等价报告。

### 不在范围内

- 不实现正式 Evaluation 指标框架；该内容属于 Day 7。
- 不引入 BM25、RRF、Rerank。
- 不制作 Streamlit 页面。
- 不实现 LangGraph。
- 不为了制造“优化有效”而篡改检索结果或只挑选不可复现样例。
- 不下载体积过大的外部文档。

## 约束

- 5 个文档应体积小、内容可读、便于进入仓库。
- PDF 必须是数字文本 PDF，不使用扫描版/OCR。
- 对比时保持 Embedding 模型、查询、top_k 等变量一致，只改变 Chunk 策略。
- 若结构感知策略没有明显优势，应如实记录，并分析数据或参数原因；不得虚构改进。
- 报告必须包含可复现命令和关键输出。
- 测试文档中不得包含 API Key、隐私数据或不允许分发的内容。
- 不对核心代码做大规模重构；仅修复验收发现的阻断性问题。

## 验证方式

### 自动化验证

执行 Day 2 相关测试，例如：

```bash
uv run pytest backend/tests -q
```

至少确保：

1. MarkdownHeadingChunker 测试通过。
2. PDFPageAwareChunker 测试通过。
3. Chunker Factory 与 Ingestion 测试通过。
4. ContextBuilder 测试通过。
5. DeepSeek Client 离线测试通过。
6. RAG Service 离线测试通过。
7. Day 1 测试无回归。

### 手工端到端验收

至少完成：

1. 一个 Markdown 分别使用 `recursive` 与 `markdown_heading` 入库。
2. 一个 PDF 分别使用 `recursive` 与 `pdf_page_aware` 入库。
3. 对每种文档至少提问一个问题。
4. 检查：
   - 答案非空；
   - 来源文件正确；
   - PDF 来源有页码；
   - Markdown 来源有章节；
   - 两种策略的检索 Chunk 可对比。
5. 保存一组“优化策略优于 Recursive”的证据；若无优势，如实记录。

## 内置文档清单
